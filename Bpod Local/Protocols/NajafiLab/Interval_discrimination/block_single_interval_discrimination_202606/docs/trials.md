# Trial And Session Computations

## Trial Types

Trial type `1` is short. Trial type `2` is long.

## Contingency

`Contingency` controls target side:

- Short-left, long-right
- Short-right, long-left

The target defines correct lick, incorrect lick, reward valve, and reward amount.

## Blocks

Blocks are generated in `GenerateTrials.m`.

Block type meanings:

- `1`: 50/50 block.
- `2`: short-majority block.
- `3`: long-majority block.

Block modes 2 and 3 always begin with one required 50/50 block. `WarmupBlockNum` adds extra leading 50/50 blocks before the remaining block logic starts. It defaults to `1` and can be set to `0`.

When `BlockNum` is `1`, every block is type `1`.

When `BlockNum` is `2`, after warmup blocks, block types `2` and `3` alternate with a random starting side.

When `BlockNum` is `3`, after warmup blocks, subsequent blocks are generated without immediate repeats.

`MostFraction` controls the majority fraction in type `2` and type `3` blocks. `BlockEdgeTrials` applies majority-biased edge trials at the start and end of each block.

## ISI

Each trial samples its ISI at trial time.

Fixed mode uses the fixed value. Random mode draws uniformly from the min/max range.

## ITI And Punish ITI

Manual mode uses the manual value.

Exponential mode draws from an exponential distribution bounded by the configured min/max values.

## Probe Types

Probe trials are generated by `GenerateProbeTrials.m`.

- `0`: normal trial.
- `1`: stimulus only, then ITI.
- `2`: servo moves in, waits for `ChoiceWindow_s`, moves out, then ITI.

## Opto And Chemo

Opto trial types are allocated by `GenerateOptoTrials.m` as a `7 x nTrials` matrix and assigned trial by trial by `GenerateOptoTrial.m`.

For full opto usage details, see `opto-control.md`.

Rows are:

- `1`: stimulus period.
- `2`: spout-in-delay period.
- `3`: spout-in period.
- `4`: pre-outcome period.
- `5`: reward period.
- `6`: post-reward period.
- `7`: punish-ITI period.

Each column is one trial. The session starts with an intended opto schedule. When a trial starts, that column is regenerated from the current GUI settings and overwritten. A non-opto trial is all zeros. On an opto trial, the current GUI period checkboxes are copied into that column, so a trial can have multiple opto periods.

`OptoMode` controls opto tagging:

- `1`: no opto trials.
- `2`: random eligible trials. Warmup blocks are excluded, and the first/last `OptoZeroEdgeTrials` trials of each non-warmup block are excluded. Because opto is assigned online, each eligible trial is sampled with probability `OptoFraction` when it starts.
- `3`: all early trials in every non-warmup block. The early window is `OptoEarlyTrials`.
- `4`: all early trials in alternating groups of blocks. After warmup blocks, the first `BlockNum` blocks are no-opto, the next `BlockNum` blocks have early opto trials, then no-opto again, and so on.

`ChemoMode` is a session-level tag. Once enabled, `ChemoTrialTypes` and `ChemoTrialType` are saved as `1` for completed trials.

## Opto Hardware Timing

`OptoControl.m` creates seven Bpod global timers, all driving `PWM1`.

- Stimulus opto starts at `PreStimDelay` onset and lasts until stimulus-play offset. It covers pre-stimulus delay plus stimulus playback only on normal, stimulus-only probe, and servo-only probe trials.
- Spout-in-delay opto starts at `SpoutInDelay` onset and lasts for `SpoutInDelay_s`.
- Spout-in opto starts during `ChoiceWindow`, `ProbeChoiceWindow`, or naive `WaitForCorrectLick`.
- Pre-outcome opto starts at `PreOutcomeDelay` or `PreOutcomeDelayPunish` onset and lasts for `PreOutcomeDelay_s`.
- Reward opto starts at `Reward` onset and lasts for the valve-open reward duration.
- Post-reward opto starts at `PostRewardDelay` onset and lasts for `PostRewardDelay_s`.
- Punish-ITI opto starts at `PunishITI` onset and lasts through `PunishITI` offset.

All opto timers are cancelled and `PWM1` is set low at trial start, servo-out, and ITI. `PunishITI` can then start its own selected opto timer. Opto is disabled in naive sessions.

Changing opto GUI settings during a pause or between trials affects the next assigned trial and all later trials. Completed trial opto columns are not rewritten.

## Outcomes

Outcome codes:

- `1`: reward.
- `2`: wrong side.
- `3`: did not choose.
- `4`: change of mind.

Reaction time is computed as:

```text
first lick after ChoiceWindow onset - ChoiceWindow onset
```

The reaction-time plot uses all three lick ports.
