# GitHub Pages Deployment

This folder is ready for GitHub Pages.

## Recommended Setup

1. Push the repository to GitHub.
2. Open the repository on GitHub.
3. Go to `Settings`.
4. Go to `Pages`.
5. Set `Source` to `Deploy from a branch`.
6. Select the branch that contains this protocol.
7. Set the folder to `/docs`.
8. Save.

GitHub will build the site from `docs/index.md`.

## Local File Structure

The documentation files are:

- `docs/index.md`
- `docs/workflow.md`
- `docs/parameters.md`
- `docs/modules.md`
- `docs/state-machine.md`
- `docs/plots-and-data.md`
- `docs/operation-checklist.md`
- `docs/deployment.md`
- `docs/_config.yml`

## Update The Site

Edit the Markdown files in `docs/`.

Commit the changes.

Push to GitHub.

GitHub Pages will rebuild the site.

## Notes

Use short headings.

Use simple sentences.

Keep file names stable so links do not break.
